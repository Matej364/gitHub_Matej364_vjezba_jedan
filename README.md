# gitHub_Matej364_vjezba_jedan
# 1. Što je Visual Studio Code i za što se najčešće koristi?
**Lagani, besplatan editor koda koji se koristi za programiranje u raznim jezicima.**
# 2. Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta?
**Explorer.**
# 3. Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe?
**U panelu Extensions (ikona kockica). Služe za dodavanje funkcionalnosti editoru.**
# 4. Kako otvoriti postojeći projekt ili mapu u VS Code-u?
**File → Open Folder.**
# 5. Što se sve može raditi u Exploreru?
**Pregledavati, otvarati, brisati, preimenovati i stvarati datoteke/mape.**
# 6. Kako možeš brzo pronaći određenu datoteku u projektu?
**Pritiskom Ctrl + P i upisivanjem imena datoteke.**
# 7. Kako možeš istovremeno gledati dvije različite datoteke u editoru?
**Klikom na Split Editor ili povlačenjem datoteke na drugu stranu editora.**
# 8. Kako pokrećeš automatsko formatiranje koda u VS Code-u?
**Shift + Alt + F ili desni klik → Format Document.**
# 9. Što je version control i zašto se koristi?
**Sustav za praćenje promjena u kodu; omogućuje povratak na stare verzije i timski rad.**
# 10. Kako se u VS Code-u otvara panel za Git (Source Control)?
**Klik na ikonu Source Control (ikona grane) u lijevom meniju.**
# 11. Što znače commit, push, pull?
**Commit: spremanje promjena lokalno.**
**Push: slanje promjena na server (GitHub).**
**Pull: preuzimanje najnovijih promjena s servera.**
# 12. Čemu služi commit poruka i kako treba izgledati?
**Opisuje što je promijenjeno. Treba biti kratka i jasna.**
# 13. Kako možeš vidjeti koje si promjene napravio u pojedinoj datoteci?
**U Source Control panelu klikneš na datoteku – prikazuje se diff.**
# 14. Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?
**Dodati datoteku (automatski se prikaže u SC).**
**Napisati commit poruku.**
**Kliknuti Commit.**
**Kliknuti Push.**
# 15. Kako se vraćaš na stariju verziju datoteke ako si pogriješio?
**Otvoriti datoteku → Timeline → odabrati stariju verziju ili koristiti Git Checkout.**
# 16. Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene?
**U Source Control panelu – prikazuje sve promijenjene datoteke.**
